We are working on a new CRM project for hospitals.
The stack should be:
- Backend: Python, Django, swagger for api docs, any other util lib when needed, for all features there should be REST route with api token auth.
- Database: PostgreSQL
Backend with database should run from docker-compose file.

Features:
- User Authentication and Authorization
- API tokens management for admin users.
- User roles
  - Admin
  - Doctor
- Modules:
 - Patient, Appointment, Lead with leads for patients, Notifications
 Create needed models in django for all these modules, there should be search endpoind with filters when it makes sense.

Write in rules.md file decisions about architecture and patterns, and what util libs are used for consistent code style.

- Frontend: React and typescript, refine lib, shadcn ui and highcharts, tailwindcss, any other util lib when needed, the frontend should be in russian.
